# Project Overview

## Purpose

**GitHub Copilot OpenAI Proxy** is a VS Code extension that exposes GitHub Copilot / VS Code Language Model API models through a local OpenAI-compatible HTTP service. This allows external tools (agent tools, coding assistants, Python scripts, LangChain, curl, etc.) to use VS Code's local language models (provided by GitHub Copilot) through the standard OpenAI API format.

## Repository Type

VS Code Extension — TypeScript, bundled with tsup.

## Main Entry Points

| Entry Point | File | Purpose |
|---|---|---|
| Extension activation | [`src/extension.ts`](src/extension.ts:12) | VS Code calls `activate()` on extension load |
| HTTP server | [`src/server.ts`](src/server.ts:7) | Express server with `/v1/models` and `/v1/chat/completions` |
| LM bridge | [`src/lmBridge.ts`](src/lmBridge.ts:3) | Translates OpenAI API calls to VS Code Language Model API |
| Sidebar UI | [`src/webview/provider.ts`](src/webview/provider.ts:5) | WebviewViewProvider for the sidebar panel |
| Webview client | [`src/webview/main.js`](src/webview/main.js:1) | Client-side JS running inside the webview |

## Runtime Entry Point

The extension entry point declared in `package.json` is `out/extension.js` (compiled from `src/extension.ts`). The `main` field in [`package.json`](package.json:33) points to this file.

## What Not to Touch Casually

- [`package.json`](package.json) `contributes` section — defines the sidebar panel registration; breaking it breaks the UI.
- [`src/webview/provider.ts`](src/webview/provider.ts:128) `_getHtmlForWebview()` — template string generates the entire webview HTML; changes here can silently break the UI.
- `server.ts` socket tracking logic (`_sockets` set) — ensures clean server shutdown; removing it causes port leaks.
- `lmBridge.ts` message mapping (`mapMessages`) — system message prepending is a deliberate workaround for VS Code API limitations.
